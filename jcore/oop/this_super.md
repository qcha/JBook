## Введение

Когда мы говорим про наследование, то у начинающих часто возникает путаница.

Давайте вспомним, что наследование — это механизм, который позволяет одному классу перенять свойства и поведение другого класса.
Есть родительский класс (базовый), есть дочерний класс (наследник).

Дочерний класс:

* Получает поля и методы родителя
* Может добавлять своё поведение
* Может переопределять поведение родителя

## Ключевое слово this

В `Java` существует ключевое слово `this` - это ссылка на текущий объект.

Зачем нужен `this`?

Отличать поля объекта от параметров:

```java
public Person(int age, String name) {
  this.age = age;
  this.name = name;
}
```

Вызывать другой конструктор этого же класса (например, со значениями по умолчанию):

```java
public Person(int age, String name) {
  this.age = age;
  this.name = name;
}

public Person(String name) {
  this(20, name);
}
```

Здесь мы объявили два конструктора класса, один из которых принимает два аргумента, а другой - только один.
В теле второго конструктора мы вызываем первый, наиболее расширенный, передавая ему значение по умолчанию `20`.

Второй конструктор по сути равносилен следующей записи:

```java
public Person(String name) {
  this.age = 20;
  this.name = name;
}
```

---

Если вам потребовалось внутри класса вызвать конструктор, например, более расширенный, то делать это надо именно через `this`:

```java
class Employee extends Person {
    protected String address;

    public Employee(int age, String name, String address) {
        super(age, name);
        this.address = address;
    }

    public Employee(int age, String name) {
        this(age, name, "EMPTY");
    }
}
```

В примере выше определены два конструктора, где внутри более короткого вызывается конструктор с наибольшим количеством аргументов.

Этот прием используется, например тогда, когда вам необходимо для каких-то полей сделать значения по умолчанию.

## Ключевое слово super

Теперь представим, что мы хотим создать новый класс, на базе `Person`, и назовем его `Employee`.

```java
public class Employee extends Person {
  private int salary;
  // ...
}
```

И здесь `this` уже будет помогать в доступе до полей и методов класса `Employee`. Поскольку `Employee` наследуется от `Person`, у объекта есть и родительская часть.

Но как нам "достучаться" до родительской части объекта?

Здесь нам и пригодится ключевое слово `super`.

Ключевое слово `super` — это ссылка на родительскую часть текущего объекта.

Например, пусть это конструктор класса `Person`:

```java
public Person(int age, String name) {
  this.age = age;
  this.name = name;
}
```

Тогда при создании класса `Employee` нам необходимо будет вызвать этот конструктор.

```java
public class Employee extends Person {
  private int salary;
  
  public Employee(int age, String name, int salary) {
    super(age, name);
    this.salary = salary;
  }
}
```

Если задуматься, то это логично: создавая дочерний класс, вы опираетесь на структуру родительского.
Подобно тому, как нельзя построить дом без фундамента, нельзя создать объект-наследник без корректной инициализации родительской части.

Точно также как и с `this`, с помощью `super` можно достучаться до полей или методов родителя:

```java
super.toString();
```

Но доступ может быть ограничен с помощью [модификаторов доступа](./encapsulation.md#модификаторы-доступа).

## Связь с java.lang.Object

В `Java` все классы неявно являются наследниками `java.lang.Object`.
Поэтому при создании объекта происходит вызов конструктора родительского класса с помощью `super()` — даже если это явно не указано.

При этом, даже если вы не создали ни одного конструктора класса, то `Java` все равно создаст пустой конструктор по умолчанию и вызовет там `super()`.

Ведь мы создаем объект класса, родительский класс которого `java.lang.Object`.

## Ограничения

И `this`, и `super` — это нестатические переменные, соответственно их нельзя использовать в статическом контексте.

```java
class Test {
    static void print() {
        System.out.println(this); // ошибка компиляции
    }
}
```

Переменной `this` или `super` нельзя присвоить новое значение, что тоже логично.

```java
this = new Object(); // ошибка!
```

Вызов конструктора `this()` или `super()` обязан быть первой строкой:

```java
class Person {
    Person(int age) {}
}

class Employee extends Person {
    Employee() {
        System.out.println("Hello");
        super(10); // ошибка компиляции
    }
}
```

## Заключение

Для закрепления:

| Ключевое слово | Описание                                               |
|----------------|--------------------------------------------------------|
| this           | Ссылка на текущий объект                               |
| super          | Ссылка на родительскую часть объекта                   |
| this()         | Вызов конструктора текущего класса                     |
| super()        | Вызов конструктора родительского класса                |
